The input is a natural-language message specifying what the prompt should be about and any details to include.
The input can relate to any domain (e.g., programming, math, creative writing, business, science) and use domain-specific terminology.
The input may provide goals, audience, tone, constraints, context, examples, or specific instructions to guide the prompt.
The input may include text to be incorporated verbatim, such as definitions, requirements lists, or sample content.
The input may specify chosen additions by referencing their uppercase-alpha labels provided in the prior message (e.g., A, B, C).
The input may select any subset of additions, including none or all, and may quote or paraphrase the option wording.
The input may provide answers to any of the previously posed clarifying questions in free-form text.
The input may omit answers to some or all questions without invalidating the input.
The input may supply new information or modifications in subsequent turns to refine or change the prompt.
The input may override earlier details or redirect the topic, and such changes remain valid.
The input is plain text and may include structured lists, greetings, bullet points, code snippets, or file content presented as text.
The input has no length constraints and may range from a brief phrase to extensive multi-paragraph specifications.
