The input is a user message string intended for academic learning support or homework help across general knowledge subjects.
The input may also be a greeting, preference, acknowledgement, or other conversational text without explicit academic content.
The input can be an academic question or topic request, including programming, science, humanities, social sciences, arts, or general knowledge.
The input can be a math homework problem requiring a multi-step or mathematical response, expressed with numbers, variables, equations, or word problems.
The input can be a non-math homework question asking about concepts that require reasoning, comparison, or argumentation.
The input can be a simple factual question such as dates, names, places, definitions, or translations.
The input can specify a learning goal, preferred subtopics, pacing, difficulty, or request to create or revise a learning plan.
The input can request or choose learning activities such as debates, role-play, riddles, vocabulary clues, or scenario-based quiz questions.
The input can provide answers to quiz questions, partial solutions to problems, or attempts that demonstrate current understanding.
The input can accept or decline offers to proceed, to solve step-by-step, or to receive more practice problems.
The input can request a summary of key points or a short quiz to check understanding.
The input can include personal interests or context to help connect academic content to the user's life and motivations.
The input can include portrayal of protected groups and may explicitly instruct use of slang, dialects, or accents.
The input can be a non-academic request or topic (e.g., planning a trip or making a purchase) that is within the broader input domain even if not supported.
The input can be a string containing the content of a file and will be treated as plain text.
The input may be short or long, and no explicit length limit is specified.
The input must be provided as natural-language text (and may include mathematical notation) in a single message without requiring special formatting.
The input should contain only textual content and avoid non-textual binary data or file metadata.
